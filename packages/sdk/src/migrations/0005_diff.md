# Diff for migration 0005

Migration file: `cms-migrations.ts` — `migration_0005_skill_usage_procs`

## Table Changes

None.

## New Indexes

```sql
CREATE INDEX IF NOT EXISTS idx_<schema>_events_skill_signals
    ON <schema>.session_events (session_id, created_at DESC)
    WHERE event_type IN ('skill.invoked', 'learned_skill.read');
```

Single partial B-tree covering both rare skill-signal event types. The
predicate excludes ~99% of `session_events` rows (assistant deltas, tool
execution events, etc.) so the index stays small. Per-session and tree
queries become single-seek + range scans; fleet query uses it for the
time-window filter. Index write cost is paid only when one of the two
event types fires.

## New Event Types Recognized

- `skill.invoked` — already emitted by the Copilot SDK, captured by
  `ManagedSession`. Payload: `{ name, path, pluginName?, pluginVersion?,
  description?, ... }`. No code change needed for this side.
- `learned_skill.read` — **new**, emitted by the `read_facts` tool
  wrapper in `facts-tools.ts` whenever the call touches `skills/...`.
  Payload: `{ name, scope, matchCount, limit, callerSessionId,
  callerAgentId? }`.

The migration treats them uniformly: the new procs return a `kind` column
(`'static'` | `'learned'`) derived from `event_type`. `name` is read from
`data->>'name'` for both event types.

## Function Changes

### `cms_get_session_skill_usage(p_session_id, p_since)` — new

```diff
+ RETURNS TABLE (
+     kind           TEXT,
+     name           TEXT,
+     plugin_name    TEXT,
+     plugin_version TEXT,
+     invocations    BIGINT,
+     first_used_at  TIMESTAMPTZ,
+     last_used_at   TIMESTAMPTZ
+ )
```

Aggregates `session_events` rows in `('skill.invoked',
'learned_skill.read')` for a single session, grouped by `(kind, name,
plugin_name, plugin_version)`. Uses the new partial index. Optional
`p_since` cutoff for time-windowed reads.

### `cms_get_session_tree_skill_usage(p_session_id, p_since)` — new

```diff
+ RETURNS TABLE (
+     session_id     TEXT,
+     agent_id       TEXT,
+     kind           TEXT,
+     name           TEXT,
+     plugin_name    TEXT,
+     plugin_version TEXT,
+     invocations    BIGINT,
+     first_used_at  TIMESTAMPTZ,
+     last_used_at   TIMESTAMPTZ
+ )
```

Same as the per-session form, but returns one row per
`(session_id, kind, name, plugin)` across the recursive spawn tree
rooted at `p_session_id`. Recursion walks `sessions.parent_session_id`
and carries `agent_id` for display. Roll-up to the tree summary happens
in the management client.

### `cms_get_fleet_skill_usage(p_since, p_include_deleted)` — new

```diff
+ RETURNS TABLE (
+     agent_id       TEXT,
+     kind           TEXT,
+     name           TEXT,
+     plugin_name    TEXT,
+     plugin_version TEXT,
+     session_count  BIGINT,
+     invocations    BIGINT,
+     last_used_at   TIMESTAMPTZ
+ )
```

Fleet-wide aggregate joining `session_events` to `sessions` for
`agent_id`. `p_include_deleted` controls whether soft-deleted sessions
contribute. Always pass a `p_since` lower bound for the default UI view.
