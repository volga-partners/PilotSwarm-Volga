## Stored Procedures

### `cms_get_user_stats_by_model(include_deleted, since)` — new

Adds a read-only CMS aggregate for the stats pane `Users` sub-tab.

The procedure groups `session_metric_summaries` by:

- owner classification: `user`, `system`, or `unowned`
- owner identity fields from `users` when present
- model

It returns per-bucket session IDs plus:

- session count
- token input/output/cache totals
- snapshot size totals
- dehydration/hydration/lossy-handoff totals
- earliest session timestamp

Runtime orchestration history size is not stored in CMS, so the
management client enriches returned session IDs by calling the
orchestration provider and summing `historySizeBytes` into each user and
model bucket.

## Direct Table Access Guidance

Application code should continue to read CMS through stored procedures
only. The new user aggregate reads through `cms_get_user_stats_by_model`
and does not require direct access to `users`, `session_owners`, or
`session_metric_summaries`.
