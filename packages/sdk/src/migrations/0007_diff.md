## New Stored Procedures

### `cms_get_session_tree_stats_by_model(p_session_id TEXT)` — new

Per-model breakdown across the spawn tree rooted at a session. Mirrors
`cms_get_fleet_stats_by_agent`'s row shape (minus the `agent_id`
dimension) so the TUI/portal "By Model" card can render uniformly for
both the fleet view and the per-session tree view.

| Column | Type | Meaning |
|---|---|---|
| `model` | TEXT | The model column from `session_metric_summaries`, or `'(unknown)'` when null |
| `session_count` | INT | Number of sessions in the tree on this model |
| `total_tokens_input` | BIGINT | Sum of `tokens_input` |
| `total_tokens_output` | BIGINT | Sum of `tokens_output` |
| `total_tokens_cache_read` | BIGINT | Sum of `tokens_cache_read` |
| `total_tokens_cache_write` | BIGINT | Sum of `tokens_cache_write` |
| `total_snapshot_size_bytes` | BIGINT | Sum of `snapshot_size_bytes` |

Uses the same recursive-descendant CTE as
`cms_get_session_tree_stats(session_id)` so the two stay in sync.
Ordered by `total_tokens_input DESC, model` so the heaviest model
appears first.

`CREATE OR REPLACE` is sufficient (the function does not exist yet, so
no return-type-change gotcha).

## No new indexes

`session_metric_summaries(parent_session_id)` already exists from
migration 0002 and powers the recursive descent.
