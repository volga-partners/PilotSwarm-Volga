# Diff for migration 0006

Migration file: `cms-migrations.ts` — `migration_0006_fleet_cache_columns`

## Table Changes

None.

## Function Changes

> Both procs are dropped and recreated rather than `CREATE OR REPLACE`d
> because PostgreSQL refuses to replace a function whose `RETURNS TABLE`
> shape changes. The migration uses `DROP FUNCTION IF EXISTS` for
> idempotency.

### `cms_get_fleet_stats_by_agent` — replaced

```diff
  RETURNS TABLE (
      ...,
      total_tokens_input          BIGINT,
-     total_tokens_output         BIGINT
+     total_tokens_output         BIGINT,
+     total_tokens_cache_read     BIGINT,
+     total_tokens_cache_write    BIGINT
  )
```

Aggregates `tokens_cache_read` / `tokens_cache_write` columns that
already exist on `session_metric_summaries`. Pure read-side fix — the
data was being collected per session but never surfaced through the
fleet aggregate.

### `cms_get_fleet_stats_totals` — replaced

```diff
  RETURNS TABLE (
      session_count                INT,
      total_snapshot_size_bytes     BIGINT,
      total_tokens_input           BIGINT,
      total_tokens_output          BIGINT,
+     total_tokens_cache_read      BIGINT,
+     total_tokens_cache_write     BIGINT,
      earliest_session_created_at  TIMESTAMPTZ
  )
```

Same change at the totals level, joined to the same data source.

## Caller Updates

`PgSessionCatalogProvider.getFleetStats()` (in `cms.ts`) projects the
two new columns into `FleetStats.totals.totalTokensCacheRead` /
`totalTokensCacheWrite` and `FleetStats.byAgent[].totalTokensCacheRead`
/ `totalTokensCacheWrite`. Existing per-session and tree procs already
returned cache counts; this migration brings the fleet view to parity.

The TypeScript layer additionally derives a `cacheHitRatio` field per
summary using a single shared helper (`computeCacheHitRatio`) so every
surface (per-session, tree, fleet) reports the same value with the same
null-handling. Definition lives only in `cms.ts`; nothing extra in SQL.
